Enum ModelParsingOption

Enum ModelParsingOption defines options that may change the parser's behavior.

Namespace: DTDLParser
Assembly: Microsoft.Azure.DigitalTwins.Parser.dll
Syntax
[Flags]
public enum ModelParsingOption
Remarks

These values can be set via the Options property. To specify more than one option, use bitwise OR to combine option values.

Fields

Name Description
AllowNonDtmiContexts

The AllowNonDtmiContexts option tells the parser to continue parsing if it encounters a remote context specifier that is not a DTMI.
AllowUndefinedExtensions

The AllowUndefinedExtensions option tells the parser to continue parsing if it encounters a reference to an extension that cannot be resolved. Specifically, if no DtmiResolver is specified, the parser will not throw an exception on encountering an extension reference, and if a DtmiResolver is specified but returns null when called, the parser will not throw an exception if it had called the resolver to resolve an extension reference.
This option has no effect on references to external definitions that are not extensions, such as references via "contents" or "extends" properties; these definitions are always required.
None

No option.
ParseAllowsIdReferenceSyntax

The ParseAllowsIdReferenceSyntax option tells the parser while parsing a submitted model to treat a JSON object containing nothing but an @id property as though it were a JSON string whose value is the value of the @id property. For example, "extends": { "@id&quot;: "dtmi:foo:bar;1" } is treated in the same manner as "extends": "dtmi:foo:bar;1"
ParseAllowsReservedIds

The ParseAllowsReservedIds option tells the parser while parsing a submitted model to permit defining DTDL elements whose identifiers have reserved prefixes.
ParseIgnoresElementsWithAutoIDsAndDuplicateNames

The ParseIgnoresElementsWithAutoIDsAndDuplicateNames option tells the parser while parsing a submitted model to ignore any duplicate names and accept the first one in the list, unless the element has a user-assigned ID. This option is present to preserve backward-compatibility with a bug that incorrectly allowed invalid models.
PermitAnyTopLevelElement

The PermitAnyTopLevelElement option tells the parser to place no restrictions on @type of the top-level object in each JSON text submitted to ParseAsync(). Without this option, only partition-type objects and rootable-type objects are permitted at the top level.
PreserveElementAliases

The PreserveElementAliases option tells the parser not to replace standard elements that are tagged as aliases with the elements to which the aliases map.
ResolveAllowsIdReferenceSyntax

The ResolveAllowsIdReferenceSyntax option tells the parser during DTMI resolution to treat a JSON object containing nothing but an @id property as though it were a JSON string whose value is the value of the @id property. For example, "extends": { "@id&quot;: "dtmi:foo:bar;1" } is treated in the same manner as "extends": "dtmi:foo:bar;1"
ResolveAllowsReservedIds

The ResolveAllowsReservedIds option tells the parser during DTMI resolution to permit defining DTDL elements whose identifiers have reserved prefixes.
ResolveIgnoresElementsWithAutoIDsAndDuplicateNames

The ResolveIgnoresElementsWithAutoIDsAndDuplicateNames option tells the parser during DTMI resolution to ignore any duplicate names and accept the first one in the list, unless the element has a user-assigned ID. This option is present to preserve backward-compatibility with a bug that incorrectly allowed invalid models.
